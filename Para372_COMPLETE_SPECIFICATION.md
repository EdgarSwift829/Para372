## 1. 概要

### 1.1 EAの特徴

Para372は「372手法」（3本のパラボリックSAR）を改良したトレンドフォロー型の自動売買システムです。

**主な特徴:**
- ✅ 15分足、1時間足、4時間足の3本のパラボリックSARを使用
- ✅ ATRベースの動的ストップロス
- ✅ マルチタイムフレームトレンドフィルター
- ✅ 経済指標発表時の自動停止機能（v1.2～）
- ✅ 負け後の自動休止機能
- ✅ 複利運用対応（資金増減に応じたロット自動調整）

### 1.2 動作環境

| 項目 | 要件 |
|------|------|
| プラットフォーム | MetaTrader 5 |
| 時間足 | 15分足（推奨） |
| 通貨ペア | USD/JPY（推奨）、EUR/USD、GBP/USD、AUD/USD |
| 最小資金 | USD/JPY: 4万円～、推奨10万円以上 |

### 1.3 期待パフォーマンス

| 指標 | 目標値 |
|------|--------|
| プロフィットファクター | 1.3～1.5 |
| 勝率 | 65～70% |
| 最大ドローダウン | 10%以下 |
| 平均トレード頻度 | 1日0～2回 |

## 2. 基本戦略

### 2.1 372手法とは

3本の異なる時間足のパラボリックSARを使用し、すべてが同じ方向を示した時のみエントリーする手法です。

### 2.2 パラボリックSAR設定

| 時間足 | Step | Maximum | 役割 |
|--------|------|---------|------|
| 15分足 | 0.001 | 0.2 | 短期トレンド検出 |
| 1時間足 | 0.01 | 0.2 | 中期トレンド検出 |
| 4時間足 | 0.035 | 0.2 | 長期トレンド検出 |

### 2.3 エントリー条件

#### 買いシグナル
```
価格 > 15分足SAR かつ
価格 > 1時間足SAR かつ
価格 > 4時間足SAR
```

#### 売りシグナル
```
価格 < 15分足SAR かつ
価格 < 1時間足SAR かつ
価格 < 4時間足SAR
```

### 2.4 決済方法

**ドテン方式:**
- 逆シグナルが出た時点で決済
- 即座に反対方向のポジションをオープン

**例:**
1. 買いポジション保有中
2. 売りシグナル発生
3. 買いポジション決済 → 即座に売りポジションオープン

## 3. 資金管理とロット計算

### 3.1 基本方針

**固定リスク方式:**
- 1トレードあたり口座残高の2%をリスク
- 資金増減に応じてロットサイズを自動調整
- 最小ロット未満の場合はエントリー見送り

### 3.2 ストップロス計算

```
SL幅 = ATR(14) × 1.5
```

**ATR（Average True Range）:**
- 期間: 14
- 倍率: 1.5（調整可能）
- ボラティリティに応じて自動調整

### 3.3 テイクプロフィット計算

```
TP幅 = SL幅 × リスクリワード比率(1.5)
```

### 3.4 ロット計算式

```
1. リスク金額 = 口座残高 × 2%
2. SL幅（ポイント） = ATR × 倍率 ÷ Point
3. 1ロットあたりリスク = SL幅 × ティック価値
4. ロット = リスク金額 ÷ 1ロットあたりリスク
5. ロットステップに丸める
6. 最小・最大ロット範囲内に収める
```

### 3.5 ロット計算の実例

**条件:**
- 口座残高: 100,000円
- リスク: 2%
- ATR: 50pips
- SL幅: 75pips

**計算:**
```
リスク金額 = 100,000 × 0.02 = 2,000円
1ロットあたりリスク = 75pips × 100円 = 7,500円
ロット = 2,000 ÷ 7,500 = 0.0267
→ 0.02ロット（ステップに丸める）
```

## 6. 停止機能

### 6.1 負け後の休止機能

**目的:** 連敗時のメンタル保護とシステムリセット

#### 設定パラメータ

| パラメータ | デフォルト値 | 説明 |
|-----------|------------|------|
| UseLossBreak | true | 休止機能の有効/無効 |
| LossBreakCount | 5 | 連敗回数トリガー |
| LossBreakUnit | "Hours" | 単位（Hours/Bars/Days） |
| LossBreakPeriod | 4 | 休止期間 |

#### 動作フロー

```
1. 負けトレード発生 → 連敗カウンター+1
2. 連敗カウンター = 5 → 休止期間開始
3. 4時間の取引停止
4. 4時間経過 → 自動再開
5. 勝ちトレード発生 → カウンターリセット
```

#### 休止期間の単位

**Hours（時間）:**
```
LossBreakPeriod = 4  // 4時間
```

**Days（日数）:**
```
LossBreakPeriod = 1  // 1日
```

**Bars（バー数）:**
```
LossBreakPeriod = 16  // 16バー（15分足で4時間）
```

### 6.2 月末・月初停止機能

**目的:** 流動性低下と値動き変則化の回避

#### 停止日

| タイミング | 停止対象 |
|-----------|---------|
| 月末 | その月の最終営業日 |
| 月初 | 翌月の1日目 |

#### 設定

```
UseMonthEndStart = false  // デフォルトOFF（オプション）
```

**有効にする場合:**
- 月末月初の異常な値動きを避けたい
- 保守的な運用をしたい

**無効のまま使う場合:**
- トレード機会を最大化したい
- 標準的な運用

## 7. 経済指標停止機能

### 7.1 概要（v1.2新機能）

重要な経済指標発表の前後は相場が急変動するため、指定時間帯の取引を自動的に停止します。

### 7.2 停止される取引

- ✅ 新規エントリー
- ✅ ドテンエントリー（決済は実行される）

### 7.3 停止されない操作

- 既存ポジションの保有継続
- SL/TPによる自動決済

### 7.4 パラメータ設定

```cpp
UseNewsFilter = true                // 経済指標フィルター使用
NewsStopMinutesBefore = 60         // 指標前60分から停止
NewsStopMinutesAfter = 60          // 指標後60分まで停止
StopOn_High_Impact = true          // 高重要度指標で停止
StopOn_Medium_Impact = false       // 中重要度指標で停止
```

### 7.5 重要度の定義

| 重要度 | 説明 | 例 |
|-------|------|-----|
| 3（高） | 市場に大きな影響を与える | 雇用統計、政策金利、GDP |
| 2（中） | 一定の影響がある | CPI、小売売上高、PMI |
| 1（低） | 影響は限定的 | 在庫指数、中古住宅販売 |

### 7.6 経済指標の登録方法

#### コード内で直接登録（推奨）

`LoadNewsEvents()`関数を編集:

```cpp
void LoadNewsEvents()
{
    ArrayResize(newsEvents, 50);
    newsEventsCount = 0;
    
    // 今週の重要指標を登録
    AddNewsEvent("2026.02.17 22:30", "USD", "米国雇用統計", 3);
    AddNewsEvent("2026.02.18 04:00", "USD", "FOMC議事録", 3);
    AddNewsEvent("2026.02.19 16:30", "EUR", "ECB政策金利", 3);
    AddNewsEvent("2026.02.20 22:30", "USD", "新規失業保険申請", 2);
    
    Print("📰 経済指標データ読み込み完了: ", newsEventsCount, "件");
}
```

#### 毎週の更新手順

1. **ForexFactoryで確認**
   - https://www.forexfactory.com/calendar にアクセス
   - 今週の重要指標をチェック

2. **MT5サーバー時間に変換**
   - ForexFactoryはGMT表示
   - MT5サーバー時間 = GMT + ブローカーのオフセット
   - 例: GMT+2（夏時間）の場合、2時間追加

3. **コードに追加**
   - `LoadNewsEvents()`関数を編集
   - 再コンパイル（F7キー）
   - EAを再起動

#### フェーズ1: デモ運用（3～6ヶ月）
- デモ口座でEAをテスト
- バックテスト結果との乖離を確認
- パラメータの微調整

#### フェーズ2: 少額リアル運用
- **開始条件**: デモで6ヶ月連続プラス収支
- **初期資金**: 10万円～
- **月間投資額**: 最大8万円

#### フェーズ3: 本格運用
- フォワードテスト成功後
- 資金を段階的に増額

## 11. トラブルシューティング

### 11.1 エラー: 「インジケーターの作成に失敗しました」

**原因:** MT5が必要なデータをダウンロードしていない

**解決策:**
1. 該当する通貨ペアのチャートを開く
2. 1時間足、4時間足のチャートを開いて履歴をダウンロード
3. EAを再起動

### 11.2 エラー: 「注文が拒否されました」

**原因:** ブローカーの制限またはロットサイズの問題

**解決策:**
1. RiskPercentを下げる（例: 1.0%）
2. 口座残高を確認
3. ブローカーの最小ロットを確認

### 11.3 ポジションが開かない

**原因:** トレンドフィルターが厳しすぎる

**解決策:**
1. Use4HTrendをfalseに設定
2. Use1HTrendをfalseに設定
3. UseDailyTrendも一時的にfalseにしてテスト

### 11.4 経済指標が登録されていない

**表示されるメッセージ:**
```
⚠️ 経済指標が登録されていません
   手動でAddNewsEvent関数を使用して登録してください
```

**対処法:**
- `LoadNewsEvents()`関数内に`AddNewsEvent()`を追加
- 再コンパイル（F7キー）

### 11.5 指標時刻が合わない

**確認事項:**
1. MT5のサーバー時間を確認
   ```cpp
   Print("現在のサーバー時間: ", TimeCurrent());
   ```

2. ForexFactoryの時間をサーバー時間に変換
   ```
   GMT時間 + ブローカーオフセット = サーバー時間
   ```

3. 夏時間・冬時間の切り替えに注意

## 14. FAQ

### Q1: 最低いくらあればEAを動かせますか？

**A:** USD/JPYなら4万円が最低ラインですが、**10万円以上を推奨**します。4万円だと高ボラ時にエントリーできない場合があります。

### Q2: 資金が少ないとどうなりますか？

**A:** EAが自動的にエントリーを見送ります。エラーにはなりませんが、トレード機会を逃すことになります。

### Q3: 資金が増えたら自動的にロットも増えますか？

**A:** はい、自動的に増えます。資金が2倍になれば、エントリーロットもほぼ2倍になります。

### Q4: 1日に何回くらいトレードしますか？

**A:** 平均して1日0～2回程度です。相場状況によって変動します。

### Q5: デモ運用はどれくらい必要ですか？

**A:** 最低3ヶ月、推奨は6ヶ月です。バックテスト結果との乖離を確認するためです。

### Q6: 複数の通貨ペアで同時運用できますか？

**A:** はい、できます。ただし、各通貨ペアで別々のマジックナンバーを設定してください。

### Q7: 経済指標の更新を忘れたらどうなりますか？

**A:** 古い指標データでフィルタリングされるため、実際の指標時に停止しない可能性があります。毎週月曜日に更新することを推奨します。

### Q8: VerboseLogは常にONにすべきですか？

**A:** 学習段階やデバッグ時はON、本番運用で安定したらOFFにすることを推奨します。

### Q9: バックテストとフォワードテストで結果が違います

**A:** 以下の要因が考えられます：
- スプレッドの違い
- スリッページ
- 経済指標フィルターの有無
- ブローカーの約定方式

### Q10: SL/TPに到達する前に決済されることはありますか？

**A:** はい、ドテンシグナルが出た場合は即座に決済されます。
